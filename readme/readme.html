Image Server API

General notes:

  • Clients will have an apiKey. This will allow tracking, in order to:
    1. Allow each client to have different 'default' settings, such as image size
    2. Ensure fresh images are served, not duplicates of previously served images.
  • Clients receive either of two types of responses:
    1. A link to an image file, which will be on this server, or
    2. The image file itself

Methods

Clients can use either GET or POST to request images.

Options

Option Type Default Value Description Deployment Status
apiKey Required - An apiKey will be provided to clients In Development
width Optional 1200 Photo width will be constrained to a maximum of this value, reducing if needed. This should be an integer. In Development
height Optional - Photo height will be constrained to a maximum of this value, reducing if needed. This should be an integer. In Development
responseType Optional link Two possible values: "link" will return json containing a link to the photo; "file" will return a jpeg file In Development
searchTerms Optional random "random" or no value will return a random image or link In Development

Three ways to display images

I want to be a picture

Link to image on this server
  1. Client issues GET or POST request
  2. Server selects or obtains image
  3. Server returns link to image on Server
  4. Client inserts link as an <image src=[link] />
Issue a GET request
  1. Client issues GET or POST request
  2. Server selects or obtains image
  3. Server returns link to image on Server
  4. Client uses link to issue GET request directly on the image file
  5. Server returns image file
  6. Client saves image to disk, uses image locally

Oh, please please please

Server sends image file, not link
  1. Client issues GET or POST request
  2. Server selects or obtains image
  3. Server returns the image file itself
  4. Client saves image to disk, uses image locally
